# Apple / iOS Track

> For developers working on KeyFlo or other Apple platform projects.

---

## Overview

Apple development at BlocLabs centers on **Swift** and **SwiftUI** for iOS/macOS. Our primary iOS project is **KeyFlo** — a non-custodial payment keyboard extension.

---

## The Guide

We maintain a comprehensive Apple development reference:

**📚 [Apple Developer Guide](https://github.com/agent67-atlas/apple-developer-guide)**

Covers:
- **Swift Language** — modern Swift 5.9+, concurrency, protocols, macros
- **SwiftUI** — state management, navigation, @Observable, animations
- **EventKit** — calendar events, reminders, alarms
- **Human Interface Guidelines** — design principles, platform patterns, accessibility
- **Design Resources** — SF Symbols, fonts, UI kits, Icon Composer
- **Key Frameworks** — SwiftData, CloudKit, StoreKit 2, WidgetKit, App Intents
- **Developer Program** — certificates, TestFlight, App Store distribution
- **Keyboard Extensions** — constraints, architecture (KeyFlo-specific)
- **Mental Models** — engineering and design thinking for world-class Apple development

---

## Getting Started

1. Read the [Mental Models](https://github.com/agent67-atlas/apple-developer-guide/blob/main/docs/mental-models.md) first — the thinking frameworks
2. Complete [Swift Language](https://github.com/agent67-atlas/apple-developer-guide/blob/main/docs/swift.md) — the language fundamentals
3. Work through [SwiftUI](https://github.com/agent67-atlas/apple-developer-guide/blob/main/docs/swiftui.md) — the UI framework
4. Read [HIG](https://github.com/agent67-atlas/apple-developer-guide/blob/main/docs/hig.md) — design is not optional

---

## Prerequisites

- macOS with Xcode installed (full app, not just CLI tools)
- Apple Developer account (for device testing)
- Complete Phase 1-3 of this onboarding program first
